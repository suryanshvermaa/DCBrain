# Coding Standards

- **TypeScript Rules:** Strict mode enabled, no `any` types.
- **Naming Conventions:** camelCase for variables/functions, PascalCase for classes/React components, UPPER_SNAKE_CASE for constants.
- **Folder Structure:** Feature-based grouping (e.g., `src/modules/auth`).
- **Error Handling:** Centralized Express error handler, custom error classes (e.g., `NotFoundError`).
- **Comments & Logging:** Use JSDoc for complex logic. Use Winston for backend logging.
- **Commits:** Follow Conventional Commits format (`feat:`, `fix:`, `chore:`).
