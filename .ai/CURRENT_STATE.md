- **Status:** Document processing, hybrid RAG search, Chat Interface, Compliance Engine, Schedule Risk Analysis, Dashboard, Procurement, RFIs, Knowledge Graph, AI Agent Framework, Simulation Engine, real-time WebSocket notifications, system audit logs, **NCRs, Inspection Intelligence, Commissioning Copilot, Change Orders, and Quality Dashboard** are successfully implemented.
- **Route Contract:** The canonical backend API prefix is `/api/v1`; auth routes are `/api/v1/auth/register`, `/api/v1/auth/login`, `/api/v1/auth/refresh`, and `/api/v1/auth/me`. Frontend API clients must use the same prefix.
- **Auth Rate Limiting:** Disabled in `APP_ENV=development` so local registration/login loops do not lock out the developer browser. Kept enabled outside development.
- **Document Upload:** Backend now exposes project-scoped document upload/list/detail/download-url/delete endpoints under `/api/v1/projects/:id/documents`. Uploads validate size/count and file content, store private objects in MinIO with UUID object keys, persist metadata/status `QUEUED` in PostgreSQL, and soft-delete records without exposing direct public object URLs.
- **Project Creation UI:** The `/documents` page includes a New project modal wired to `POST /api/v1/projects`, so users with `create_projects` permission can create the first project before uploading documents.
- **Docker Frontend Runtime:** `docker-compose.yml` sets frontend `HOSTNAME=0.0.0.0` and `PORT=3000` so Next standalone binds correctly inside Docker. Fixed a critical React loading bug across all frontend pages where protected components would fire API fetches before the session token was fully refreshed, resolving "Missing access token" errors.
- **Chat & RAG:** LangGraph powers an autonomous ReAct AI agent. RAG pipeline uses ChromaDB and BAAI/bge-small-en-v1.5 embeddings for semantic retrieval. PDF export works properly with markdown cleaning.
- **Knowledge Graph:** Backend API (at `/api/v1/projects/:id/graph`) leverages Neo4j to manage dependencies, with queries updated to be fully case-insensitive via Cypher `toLower()`. Frontend UI (`/graph`) visualizes layered nodes with color-coded relationship edges and animated failure paths.
- **AI Agent Framework:** Backend module at `backend/src/modules/agents/` implements all 14 agents plus Supervisor. REST API at `/api/v1/projects/:id/agents` supports list, run, run history, and schedule configuration. BullMQ `agent-execution` worker processes async runs. Auto-triggers fire on document processing, schedule import, and procurement import. Frontend `/agents` page provides supervisor query routing, manual agent triggers, and run history detail view.
- **Database Migration Status:** Local Docker Postgres has all Task 001ŌĆō018 migrations applied, including `add_epc_intelligence` (migration `20260716070731`) creating `ncrs`, `inspections`, `commissioning_records`, `change_orders` tables.
- **Advanced EPC Intelligence (Task 018):** 4 new Prisma models (Ncr, Inspection, CommissioningRecord, ChangeOrder) + 5 enums. 5 backend modules with full CRUD, status transitions, and analytics. Quality score endpoint (composite of NCR health + inspection + commissioning pass rates). 5 frontend pages (NCRs, Inspections, Commissioning, Change Orders, Quality Dashboard). All routes mounted under `/api/v1/projects/:id/<module>`.
- **Reporting Engine:** Backend module at `backend/src/modules/reports/` generates 6 report types (daily, weekly, executive, compliance, risk, procurement) with AI-summarized sections, PDFKit PDF export, MinIO storage, and BullMQ async generation. REST API at `/api/v1/projects/:id/reports` with generate, list, detail, and download endpoints. ReportingAgent upgraded to call actual report generation. Frontend `/reports` page with generate dialog, report table, preview modal, and PDF/Markdown download. Manual UI verification completed: fixed MinIO signature mismatch by proxying PDF stream through backend API, and fixed PDFKit blank pagination bugs.
- **Schedule Risk Module:** Backend exposes 4 project-scoped endpoints under `/api/v1/projects/:id/schedule`: POST /import (P6 XML multipart), GET /activities, GET /summary, GET /imports. Parser uses `fast-xml-parser`, risk engine scores 0ŌĆō100, mitigations generated via Gemini with deterministic fallback. Frontend page at `/schedule` includes Recharts scatter heat map, filterable activities table, health indicator cards, and drag-and-drop import.
- **Dashboard Module:** Backend exposes GET `/api/v1/projects/:id/dashboard/summary` with Redis caching (5-min TTL, `?refresh=true` bypass). Aggregates: document stats, compliance score, schedule risk, procurement risk, unresolved/overdue RFIs, and activity feed. Health score computed as `0.4├ŚdocHealth + 0.3├ŚcomplianceScore + 0.3├Ś(100ŌłÆscheduleRisk)`. Frontend landing page (`/`) includes sidebar navigation, SVG health gauge, stat cards, document breakdown, compliance meter, schedule SPI meter, procurement pipeline, unresolved RFI overview widget, and activity feed.
- **Procurement Module:** Backend exposes POST `/import` (XLSX/CSV via `xlsx`), GET `/`, GET `/vendors`, and GET `/alternatives/:itemId`. Parses vendors, computes vendor performance scorecard, tracks item status. Connected to dashboard. Frontend `/procurement` lists items and alerts for items nearing `requiredOnSiteDate`.
- **RFI Module:** Backend exposes GET `/`, POST `/`, GET `/:rfiId`, PUT `/:rfiId`, POST `/:rfiId/suggest-answer`, and GET `/analytics`. Utilizes RAG pipeline to suggest answer drafts with source document citations. Integrates unresolved and overdue counters in DashboardSummary. Frontend page at `/rfis` contains list filters, master-detail layout, new RFI modal, document linking checkbox list, and review/approval resolution flow.
- **Deployment Infrastructure:** GitHub Actions CI (`.github/workflows/ci.yml`) runs lint+typecheck+test on every push. GitHub Actions CD (`.github/workflows/cd.yml`) builds Docker images to GHCR, auto-deploys to staging on `develop`, manual-approval deploy to production on `main`. `backend/Dockerfile` is 3-stage multi-stage (non-root user). `frontend/Dockerfile` hardened (non-root user, HEALTHCHECK). `docker-compose.prod.yml` for production. `nginx/nginx.conf` with SSL, security headers, rate limiting. `scripts/backup-db.sh` with 30-day rotation. `.ai/RUNBOOK.md` deployment runbook.
- **Test Suite:** 15 test suites / 60 tests ŌĆö report module 13/13 passing, agent module 9/9 passing; graph tests require Neo4j to be running locally.
- **Simulation Engine:** Backend API at `/api/v1/projects/:id/simulations` creates "what-if" scenarios (e.g. delays). Neo4j cascade traversal (`graphService.getFailurePropagation`) computes downstream blockages and total estimated cost impact. `MITIGATION_PLANNER` AI agent formulates step-by-step recovery plans formatted in Markdown. Frontend `/simulations` displays a dashboard of previous scenarios, `/simulations/new` uses the latest schedule activities for a target dropdown, and `/simulations/:simId` handles the results dashboard with properly formatted Markdown via `@tailwindcss/typography`. Completed manual verification including fixing the `Activity` Neo4j unique constraint to allow multi-schedule uploads.
- **Notifications & Audit:** Backend includes WebSocket real-time delivery (`websocket.ts`), database-persisted preferences (`notification_preferences`), and Express `auditMiddleware` recording write actions and administrative operations. Admin route `GET /api/v1/admin/audit-log` is RBAC-restricted to the ADMIN role. Frontend includes the glassmorphic `NotificationBell` header component, `/settings` page for preferences, `/activity` timeline query feed, and the secure `/admin/audit-log` table viewer.
