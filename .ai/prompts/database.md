# Database Prompt

---

## Prompt Template

```
You are modifying the database schema for DCBrain.

## Required Reading
1. .ai/DATABASE.md — Current complete schema
2. .ai/ARCHITECTURE.md — How the database fits in the system
3. .ai/CODING_STANDARDS.md — Database patterns

## Change Required
[Describe the schema change needed]

## Rules
- All changes via Prisma Migrate migrations (never manual SQL in production)
- Every Prisma migration generates an `migration.sql` with `apply` and `apply` rollback steps; review before applying
- UUID primary keys generated by database (`gen_random_uuid()`)
- All timestamps use TIMESTAMPTZ with DEFAULT NOW()
- Foreign keys include ON DELETE CASCADE where appropriate
- Add indexes for columns used in WHERE clauses and JOINs
- Use JSONB for flexible metadata fields
- Add comments to non-obvious columns
- Test migration on a copy before applying to production
- Update DATABASE.md after creating the migration

## Naming Conventions
- Tables: plural snake_case (users, documents, compliance_checks)
- Columns: singular snake_case (created_at, document_id, risk_score)
- Indexes: idx_{table}_{column} (idx_documents_project)
- Foreign keys: {column_name} referencing {table}(id)
- Migrations: Prisma generates `prisma/migrations/{timestamp}_{description}/migration.sql` automatically via `npx prisma migrate dev`

## Output
- Prisma Migrate migration directory and `migration.sql`
- Updated Prisma schema (`prisma/schema.prisma`)
- Updated DATABASE.md section
```
